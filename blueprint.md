# Word Photo Reader App Blueprint

## 1. 개요 (Overview)

이 프로젝트는 사용자가 단어가 포함된 이미지를 업로드하면, 이미지 속에서 한 단어를 자동으로 인식하여 원어민 발음으로 5번 반복해서 읽어주는 웹 애플리케이션입니다. 모바일 환경에 최적화된 반응형 디자인을 갖추고 있으며, 시각적으로 매력적이고 사용하기 쉬운 인터페이스를 제공하는 것을 목표로 합니다.

## 2. 디자인 및 스타일 (Design & Style)

- **레이아웃:** 화면 중앙에 핵심 기능을 담은 카드 형태의 UI를 배치하여 사용자가 기능에 집중할 수 있도록 합니다.
- **색상 팔레트:** 생동감 있고 활기찬 색상 조합을 사용하여 긍정적인 사용자 경험을 제공합니다.
- **타이포그래피:** 가독성 높은 폰트를 사용하고, 정보의 중요도에 따라 폰트 크기를 조절하여 명확한 정보 전달을 돕습니다.
- **시각 효과:** 카드와 버튼에 부드러운 그림자 효과를 적용하여 입체감을 주고, 사용자의 인터랙션에 시각적 피드백을 제공합니다.
- **반응형 디자인:** 모바일, 태블릿, 데스크탑 등 다양한 화면 크기에서 최적의 사용성을 제공합니다.

## 3. 기능 (Features)

- **이미지 업로드:** 사용자는 자신의 기기에서 단어가 포함된 이미지 파일(JPG, PNG 등)을 선택하여 업로드할 수 있습니다.
- **광학 문자 인식 (OCR):**
    - Tesseract.js 라이브러리를 사용하여 브라우저 내에서 이미지의 텍스트를 인식합니다.
    - 서버 없이 클라이언트 측에서 모든 처리가 이루어져 빠르고 안전합니다.
    - 이미지 내에서 **네모칸 안에 단독으로 있는 단어**와 가장 유사한 조건인, 다른 단어와 분리된 단일 단어를 우선적으로 추출합니다.
- **텍스트 음성 변환 (TTS):**
    - 웹 표준 API인 `SpeechSynthesis`를 사용하여 추출된 단어를 음성으로 변환합니다.
    - 'en-US' (미국 영어)를 기본 언어로 설정하여 원어민에 가까운 발음을 제공합니다.
- **반복 재생:**
    - 인식된 단어를 2초 간격으로 총 5번 반복하여 읽어줍니다.
    - 사용자가 단어의 발음을 충분히 듣고 학습할 수 있도록 돕습니다.
- **상태 표시:** 이미지 처리 중이거나, 단어를 찾지 못했을 때 사용자에게 명확한 상태 메시지를 보여줍니다.

## 4. 현재 진행 계획 (Current Plan)

1.  **HTML (`index.html`):**
    - 앱 제목, 설명, 이미지 업로드(`<input type="file">`), 실행 버튼(`<button>`), 상태/결과 표시 영역을 포함하는 기본 구조를 설정합니다.
    - Tesseract.js 라이브러리를 CDN을 통해 추가합니다.
2.  **CSS (`style.css`):**
    - 전체 페이지 스타일, 중앙 카드 레이아웃, 버튼, 폼 요소 등 디자인 가이드라인에 따른 스타일을 적용합니다.
    - 반응형 레이아웃을 위한 미디어 쿼리를 추가합니다.
3.  **JavaScript (`main.js`):**
    - DOM 요소에 대한 참조를 설정합니다.
    - 버튼 클릭 이벤트 리스너를 추가합니다.
    - 이미지 파일이 선택되면 Tesseract.js를 사용하여 OCR을 수행하는 비동기 함수를 구현합니다.
    - OCR 결과에서 단일 단어를 필터링하는 로직을 작성합니다.
    - `SpeechSynthesis` API를 사용하여 단어를 5회 반복 재생하는 함수를 구현합니다.
    - 오류 처리 및 사용자 피드백 로직을 추가합니다.
